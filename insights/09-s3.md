## What S3 Is
Amazon S3, which stands for Simple Storage Service, is an object storage service that lets you store and retrieve any amount of data from anywhere on the internet. It is not a file system, not a database, and not a server. It is a place to put files and get them back when you need them. Those files can be anything — HTML pages, images, videos, backups, logs, or any other type of data.

## Why S3 Exists
Before services like S3, storing files in a way that was accessible over the internet required owning and maintaining physical storage hardware. You had to buy drives, install them, configure them, back them up, replace them when they failed, and expand them when you ran out of space. S3 removes all of that. You create a bucket, upload your files, and they are stored reliably and accessibly without any hardware management on your part.

## How S3 Stores Files
S3 organizes data into three concepts: buckets, objects, and keys.

### Buckets
A bucket is a container for storing objects. Every bucket has a name that must be globally unique across all AWS accounts worldwide. No two buckets anywhere in AWS can share the same name. When you create a bucket, you choose a region where it will live. That bucket and everything in it stays in that region unless you explicitly configure replication.

### Objects
An object is any file you store in S3. An object can be as small as a single byte or as large as 5 terabytes. Every object stored in S3 is made up of the file itself and the metadata attached to it, which includes information about the file such as its size, type, and when it was last modified.

### Keys
A key is the unique identifier for an object within a bucket. It is the full path to the object within the bucket. If you upload a file called index.html to a bucket called my-website, the key for that object is index.html. If you organize your files into folders, the key includes the folder path, such as assets/images/logo.png.

## How S3 Differs From a Traditional File System
A traditional file system on a computer organizes files into a hierarchy of folders and subfolders. S3 looks similar because keys can contain slashes that resemble folder paths, but S3 does not actually have folders. Every object exists at the top level of the bucket, identified only by its key. The folder-like structure is an illusion created by the slashes in the key name.

S3 also differs from a traditional file system in how it scales. A hard drive has a fixed capacity. When it fills up, you have to buy a new one. S3 has no fixed capacity. You can store as little as nothing or as much as petabytes of data and S3 scales automatically without any action on your part.

## How S3 Connects to CloudFront
In this workshop, S3 is the origin. It is where your website files live. CloudFront sits in front of S3 and handles delivering those files to users around the world. When a user requests your website, CloudFront checks its nearest edge location for a cached copy of the files. If it does not have one, it fetches the files from your S3 bucket and caches them for future requests.

The connection between S3 and CloudFront is governed by permissions. By default, S3 buckets are private. CloudFront cannot access your files unless you explicitly grant it permission through a bucket policy that implements Origin Access Control.

## Key Terms
Amazon S3 — Simple Storage Service, an object storage service that lets you store and retrieve any amount of data from anywhere on the internet.

Object storage — a way of storing data where each file is stored as a discrete object identified by a unique key, as opposed to a file system that organizes data into a hierarchy of folders.

Bucket — a container in S3 for storing objects, with a globally unique name and a specific region where it lives.

Object — any file stored in S3, consisting of the file data and its associated metadata.

Key — the unique identifier for an object within a bucket, representing the full path to the object.

Metadata — information about an object stored alongside it in S3, such as its size, content type, and last modified date.

Global uniqueness — the requirement that every S3 bucket name must be unique across all AWS accounts worldwide, not just within your own account.
